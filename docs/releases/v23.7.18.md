# ISCY V23.7.18

ISCY V23.7.18 macht den Incident-Workflow noch runder: Die Fallakte erzeugt nun neben NIS2 auch DORA- und DSGVO-Pruefpakete, die Dashboard-Kachel fuer offene Erheblichkeitsbewertungen fuehrt direkt zur passenden Incident-Liste, und das Handbuch-PDF kann reproduzierbar aus Markdown gebaut werden.

## Neu

- DORA-Pruefpakete fuer Incidents als Markdown, HTML und PDF:
  - `/incidents/{id}/dora-export`
  - `/incidents/{id}/dora-export.html`
  - `/incidents/{id}/dora-export.pdf`
- DSGVO-Pruefpakete fuer Incidents als Markdown, HTML und PDF:
  - `/incidents/{id}/dsgvo-export`
  - `/incidents/{id}/dsgvo-export.html`
  - `/incidents/{id}/dsgvo-export.pdf`
- Entsprechende API-Endpunkte unter `/api/v1/incidents/{id}/...`.
- Incident-Detailseiten verlinken NIS2-, DORA- und DSGVO-Pakete direkt im Meldepaket-Bereich.
- Dashboard-Kachel `Erheblichkeit offen` ist klickbar und oeffnet `/incidents/?incident_filter=unassessed`.
- Incident-Register unterstuetzt den Filter `incident_filter=unassessed`.
- Neues Rust-Binary `iscy-doc-pdf` erzeugt `docs/ISCY_Handbuch.pdf` reproduzierbar aus `docs/ISCY_Handbuch.md`.
- Neuer Make-Target: `make docs-pdf`.

## Fachliche Einordnung

- DORA und DSGVO werden bewusst als Pruefpakete gefuehrt, nicht als automatisch behauptete Meldepflicht.
- DORA haengt vom Finanzsektor-/IKT-Dienstleister-Kontext und der Major-Incident-Klassifizierung ab.
- DSGVO haengt vom Personenbezug, dem Risiko fuer Betroffene und Datenschutz-Schutzmassnahmen ab.

## Verifikation

- `nix develop --command cargo fmt --manifest-path rust/iscy-backend/Cargo.toml`

- `nix develop --command cargo run --manifest-path rust/iscy-backend/Cargo.toml --bin iscy-doc-pdf -- docs/ISCY_Handbuch.md docs/ISCY_Handbuch.pdf`
