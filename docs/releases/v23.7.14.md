# ISCY V23.7.14

ISCY V23.7.14 macht Alert Operations handlicher und auditfaehiger: Alertmanager-Fallakten koennen jetzt direkt nach open, critical und resolved gefiltert werden, automatisch geschlossene Alerts koennen optional eine Root-Cause-/Lessons-Learned-Reviewpflicht ausloesen, und Grafana kann aus einem Alert-Panel direkt in die konkrete Incident-Fallakte springen.

## Neu

- `/operations/incidents/` unterstuetzt `alert_filter=open`, `alert_filter=critical` und `alert_filter=resolved`.
- Resolved Alertmanager-Fallakten werden optional mit Review-Pflicht markiert, wenn `ISCY_ALERTMANAGER_REQUIRE_RESOLUTION_REVIEW=1` oder `ISCY_REQUIRE_INCIDENT_ROOT_CAUSE_ON_RESOLVE=1` gesetzt ist und Lessons Learned fehlen.
- `iscy_operations_alertmanager_incidents_total` exportiert zusaetzlich `state="resolved"`.
- `iscy_operations_alertmanager_incident_info` exportiert pro Alertmanager-Fallakte Labels fuer Incident-ID, Titel, Severity, Status, State, Review-Pflicht und Href.
- Das Grafana-Dashboard enthaelt ein neues Table-Panel `Alert Incident Drilldown` mit Direktlink zur Fallakte.
- README, Operations-Doku, Cutover-Status, Handbuch und PDF-Handbuch sind auf `V23.7.14 / Rust 0.3.10` aktualisiert.

## Betrieb

Alert-Operations-Filter:

```text
/operations/incidents/?alert_filter=open
/operations/incidents/?alert_filter=critical
/operations/incidents/?alert_filter=resolved
```

Optionale Review-Pflicht fuer automatisch resolved Alerts:

```bash
ISCY_ALERTMANAGER_REQUIRE_RESOLUTION_REVIEW=1 ./start.sh
```
