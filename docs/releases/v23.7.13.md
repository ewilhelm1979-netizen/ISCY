# ISCY V23.7.13

ISCY V23.7.13 schliesst den Alertmanager-Lifecycle fachlich sauber ab: firing Alerts erzeugen oder aktualisieren weiterhin deduplizierte Incident-Fallakten, resolved Alerts schliessen die passende offene Fallakte automatisch. Zusaetzlich gibt es eine kompakte Alert-Operations-Ansicht und konfigurierbare Grafana-Drilldowns.

## Aenderungen

- Alertmanager-Persistenz verarbeitet resolved Alerts und setzt passende offene Alert-Fallakten auf `RESOLVED`.
- Resolved Alerts pflegen `resolved_at` aus `endsAt` oder ersatzweise `startsAt` und schreiben eine Timeline-Notiz.
- Die Webroute `/operations/incidents/` zeigt Alertmanager-Fallakten mit offenen, Triage-, kritisch offenen und resolved Zaehlern.
- Die Statusseite verlinkt direkt auf die Alert-Operations-Ansicht.
- Das Grafana-Panel `Alert Incidents` nutzt die Dashboard-Variable `iscy_base_url` statt einer hart codierten lokalen URL.
- README, Operations-Doku, Handbuch, Screenshot-Doku und Agent-Doku sind auf `V23.7.13 / Rust 0.3.9` aktualisiert.
