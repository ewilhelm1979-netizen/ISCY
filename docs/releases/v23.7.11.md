# ISCY V23.7.11

ISCY V23.7.11 haertet den Operations-Webhook fuer den produktionsnaeheren Betrieb: Alertmanager nutzt jetzt einen dedizierten technischen Operations-User statt Demo-Admin-Kontext, liest Bearer Tokens aus einer Secret-Datei und exportiert aus Alertmanager erzeugte Incident-Fallakten als eigene Prometheus-Metrik.

## Aenderungen

- Demo-Seed ergaenzt den technischen User `ops-alertmanager` mit User-ID `2`, Rolle `CONTRIBUTOR` und Tenant `1`.
- Alertmanager-Compose-Beispiel nutzt `http_config.authorization.credentials_file` und kann per `ISCY_ALERTMANAGER_TOKEN_FILE` auf eine lokale Secret-Datei zeigen.
- NixOS-Monitoring-Modul erhaelt `alertTokenFile`; das Beispiel verdrahtet User `2`, Rolle `CONTRIBUTOR` und `/etc/iscy/alertmanager-token`.
- Operations-Status exportiert `iscy_operations_alertmanager_incidents_total{state="all|open|triage|critical_open"}` fuer Alertmanager-Fallakten.
- Grafana-Dashboard enthaelt ein neues Panel `Alert Incidents`.
- README, Operations-Doku, Handbuch, Screenshot-Doku und Agent-Doku sind auf `V23.7.11 / Rust 0.3.7` aktualisiert.
