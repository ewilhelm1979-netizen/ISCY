# ISCY V23.7.8

ISCY V23.7.8 macht den Monitoring-Betrieb operativer: Prometheus, Alertmanager und Grafana koennen nun direkt ueber Repo-Artefakte gestartet oder in NixOS eingebunden werden. Zusaetzlich nimmt ISCY Alertmanager-Webhooks entgegen und Product Security bekommt ein Trend-Dashboard fuer CVE-, Evidence-, Import- und Coverage-Signale.

## Neu

- Monitoring Compose Stack unter `deploy/monitoring/docker-compose.yml`.
- Vollstaendige Prometheus-Konfiguration unter `deploy/monitoring/prometheus/prometheus.yml`.
- Grafana-Provisioning fuer Datasource und Dashboard unter `deploy/monitoring/grafana/provisioning/`.
- NixOS-Modulbeispiel unter `deploy/monitoring/nixos/iscy-monitoring.nix`.
- Alertmanager-Webhook `POST /api/v1/operations/alertmanager` mit optionalem `ISCY_ALERTMANAGER_TOKEN`.
- Product-Security-Trend-API `GET /api/v1/product-security/trends`.
- Product-Security-Web-UI um Trendpanel fuer Coverage, offene Schwachstellen, CVE-Reviews, Evidence-Luecken, Risiko-Luecken, Importvalidierung und Snapshot-Verlauf erweitert.
- README, Operations-Doku und Handbuch/PDF aktualisiert.
- Rust Backend Version auf `0.3.4` angehoben.

## Betrieb

ISCY starten:

```bash
./start.sh
```

Monitoring-Stack starten:

```bash
docker compose -f deploy/monitoring/docker-compose.yml up -d
```

Standard-URLs:

- ISCY: `http://127.0.0.1:9000/login/`
- Prometheus: `http://127.0.0.1:9090`
- Alertmanager: `http://127.0.0.1:9093`
- Grafana: `http://127.0.0.1:3000`

Product-Security-Trends:

```bash
curl -fsS -H 'x-iscy-tenant-id: 1' -H 'x-iscy-user-id: 1' \
  http://127.0.0.1:9000/api/v1/product-security/trends
```
