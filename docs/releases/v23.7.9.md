# ISCY V23.7.9

ISCY V23.7.9 verdichtet den Rust-only-Betrieb weiter: Alertmanager kann Betriebsalarme jetzt mit schreibendem Tenant-Kontext direkt als Incident-Fallakte, Evidence und Timeline-Ereignis persistieren. Product-Security-Trends stehen zusaetzlich als Prometheus-Metriken bereit und fuer NixOS gibt es neben dem Modul nun eine kleine Beispiel-Hostkonfiguration.

## Neu

- Alertmanager-Webhook `POST /api/v1/operations/alertmanager` persistiert firing Alerts optional als Incident und Evidence, wenn `x-iscy-tenant-id`, `x-iscy-user-id` und eine schreibende Rolle oder Session gesetzt sind.
- Webhook-Antwort enthaelt eine `persistence`-Zusammenfassung mit erzeugten Incidents, erzeugter Evidence, Skip-Grund und Fehlerliste.
- Prometheus-Metriken fuer Product-Security-Coverage, Importvalidierung, Trend-Signale, Snapshot-Readiness und offene Snapshot-Schwachstellen.
- NixOS-Beispielhost unter `deploy/monitoring/nixos/example-host.nix`.
- Dokumentation zu Operations Monitoring, NixOS-Start und Product-Security-Metriken aktualisiert.

## Version

- Rust Backend: `0.3.5`
- Repo-Stand: `ISCY V23.7.9`

## Start auf NixOS

```bash
./start.sh
```

Monitoring optional:

```bash
docker compose -f deploy/monitoring/docker-compose.yml up -d
```

Tenant-bezogene Metriken:

```bash
curl -fsS -H 'x-iscy-tenant-id: 1' -H 'x-iscy-user-id: 1' \
  'http://127.0.0.1:9000/api/v1/status/metrics?tenant_id=1&user_id=1'
```
